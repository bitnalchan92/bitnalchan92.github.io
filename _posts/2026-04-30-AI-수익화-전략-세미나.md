## 창작 도구 기준 생성형 AI

- 글, 대화 AI
- 이미지 AI
  - 나노바나나의 획기적이었던 점 : 이미지를 생성할 때 캐릭터의 일관성
  - 덕테잎의 장점 (글 작성 기준 가장 좋은 성능) : 한글이 깨지지 않고 이미지 생성 성능이 너무 좋아짐
- 영상 AI
- 음악 AI



## 자율 실행하는 AI 기준

- AI 에이전트 - 범용형
- 업무 자동화 AI
- 코딩 보조 AI
- 앱, 웹사이트 자동 제작 AI



## 돈을 버는데...

- 허들이 작은 툴을 사용하면 살짝 낮은 레벨 수준까진 가능하고,
- 여기에 코딩을 할 줄 아는 어느 정도의 베이스가 있다면 거기에 몇 단계 레벨이 더 올라갈 수 있을거라고 생각
- 강사님은 앱 웹사이트 자동 제작 쪽에 관심을 두고 있고 공부하고 있다고 함
- 코딩을 알 필요가 있냐? 
  - 아니, 그냥 창작의 영역에서 자기가 하고 싶은 걸 한다 하면 몰라도 된다. 
  - 강사님은 자신이 이걸 가지고 강의하면서 돈을 벌어야 하니까 필요에 의해 공부한다고 함



## 생성형 AI의 경향

- 너무 빠르게 변화가 진행되고 있고
- 빠른 유료화와 전문 AI가 계속 세분화 되고 있음
- AI는 도구가 아니라 '수익 구조'
  - 개인이 AI로 팀을 생성하고 자동화 구조를 만드는데 수익화 구조로 이어짐
- 내가 가지고 있는 콘텐츠가 곧 자산
- 모든 콘텐츠를 멀티모달로 제작해야 함
- 브랜드도 AI로 제작 생성 가능



## 왜 지금, AI 수익화인가?

- 콘텐츠 수익화
- 디지털 제품으로 판매
- 서비스 제공



## 네이버 블로그에 대해서

- 키워드 리서치
- Chat GPT 글 생성
- 이미지 생성
- SEO 최적화
- 발행
- 돈을 벌려면...
  - Question. 돈을 버는 거에 기본이 뭘까요? 
    - 사람이 와야 함.
    - 내가 500만원짜리든, 1000만원짜리든 내가 파는 걸 사줄 수 있는 찐팬이 100명만 있어도 먹고 살 수 있다.
    - 그 100명이 또 그들의 지인 10명을 데려오고, 그 10명이 또 10명을 데려오고... 
    - 이렇게 된다는 가정 하에 그러면 돈을 벌 수 있다.
    - 사람 모으기부터 시작하는 게 중요하다.
  - 자신이 가진 재능을 생각해보고 살 사람을 모으는 방법
    - 크몽, 숨고, 탈잉 등... (해외 Etsy, Gumroad), 캔바나 미리캔버스에 기여자로 등록
    - 후기를 만들어야 함 (순진하게 후기를 누가 자발적으로 쌓아 줄 거라고 생각하지 마라.)

## 내가 평소에 즐기는 걸로 수익화

- 네이버 블로그 글쓰기 -> 체험단 -> 브랜딩 -> 대행사 
- 카톡 이모티콘 제작 판매, 오픈톡방 활용 트래픽
- 웹소설 / 웹툰 스토리 제작 / 웹툰 이미지 제작
- 인터넷 신문 기사 작성 / 마케팅 홍보
- 인스타 피드, 릴스 작성 -> 체험단 -> 공구 -> 인플루언서
- 너무 멀리 있는 걸 잡으려 하지 말고, 내가 할 수 있는 것들을 기반으로 추려보기.
- 남들 못하는 것보다, 이미 할 줄 아는 걸 가지고 그것의 생산자가 되도록 해야 함

## 플레이리스트 채널 기획

- 책 볼 때 일하거나 공부할 때... 듣는 것

### AI 음원의 강점 = 내 건물

- 저작권 100% 내 소유 : 편집, 유통, 리믹스, 라이센스 다 확보 가능 ( 저작권 등록은 안됨, 우리나라의 경우 )
- 롱폼 광고 단가가 높음 : 숏폼 대비 20배
- 누적 재생 수익 : 한번 제작 => 오랫동안 수익
- 글로벌 확장성 : 언어장벽이 낮고, 전 세계 유통 가능

### 다양한 음원 수익화 방법

- 유튜브를 통해
- 유통사를 통해
  - 멜론, 지니, 스포티파이 등 음원 플랫폼 수익 분배
- 저작권료
  - 배경음악 사용료, 공연 사용료 등의 권리 수익
- 음악 라이센스 판매
  - 배경음악, 게임음악 판매 등등...
- 플랫폼
  - RouteNote
  - DistroKid
  - TuneCore
  - Pond5
  - AudioJungle / Envato Elements
  - Bandcamp

### 음악 기본 재료(건축 요소)

- 장르 / 멜로디 건축 양식 (집의 외관)
  - 재즈 / 어쿠스틱 / 트로트 / 신스팝 / K팝 / 오케스트라
- 리듬 / 템포 (생활의 속도 / 동네 분위기) : 분위기와 속도를 정하는 BPM
- 화성 / 악기 (인테리어와 가구)
  - 피아노 / 스트링 / 브라스 / 베이스
- 송 폼 구조(집의 평면도(설계도))
  - Intro => Verse => Chorus => Bridge => Outro
  - 이걸 잘 지키면서 [SUNO.AI](https://suno.com/discover)에게 시켜야 길이가 어느 정도 나온다.

### 프롬프트 황금공식

```
장르 -> 분위기/감정 -> 템포 느낌 -> 핵심 악기 -> 보컬 유무 & 톤 -> 가사 주제 / 키워드 -> 길이 / 용도
```

예시1) 로파이 힙합(공부용 BGM)

아늑하고 따뜻한 힙합, 느린 템포 느낌

공부용 BGM + 아날로그 질감 + 4분



예시2) K-pop 댄스 (훅 강조)

K-pop dance + 에너지 넘침 + 빠른 느낌 + EDM synth, punchy kick + 여성 남성 보컬 (파워풀) + 여름 페스티벌 주제 + 현대적 믹스 + 3분



- 구체적 형용사와 악기, 용도를 명시
  - 노래 제목을 넣는 건 되지만, 가수 이름을 명시하는 건 좋지 않다.
